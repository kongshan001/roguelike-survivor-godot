The version transitions the project from "polish and fix" mode to "new content" mode.

### 4.1 Pause Mastery Panel Implementation [Priority 1]

**Priority**: P1 HIGH
**Est. Lines**: ~95 (70 new in hud_mastery_panel.gd + 25 in hud.gd)
**Blocker**: None. Spec complete since R26.
**Spec**: `docs/superpowers/specs/pause-mastery-panel.md`

**Implementation scope**:

| Component | File | Lines | Description |
|-----------|------|-------|-------------|
| build_pause_panel() | hud_mastery_panel.gd | ~45 | Create 320x480 panel with 7 weapon rows, progress bars, Resume/Quit |
| _input() Escape handler | hud.gd | ~25 | Toggle pause, show/hide panel overlay |
| Integration tests | test file | ~22 | Guard tests (3 risky) will flip to passing |

**Key design decisions (from R26 spec)**:
- Tier-relative progress bars (not total progress)
- Quit button has no keyboard shortcut (prevent accidental exit)
- Panel uses PROCESS_MODE_ALWAYS for button interaction while paused
- Status header shows time/wave/gold/level

**Why first**: 22 tests already exist (19 passing, 3 risky guards). Completing this closes the last v1.0.3 UI gap and recovers the 0.2 score drop from risky tests.

## 10. Decision Records

| Decision | Why | Alternative Considered |
|----------|-----|----------------------|
| Pause panel first (Phase 1) | 22 tests already exist, 3 risky guards flip to passing. Quickest path to recovering score | Do weapon behaviors first (higher player impact but longer feedback cycle) |
| Phase C before Phase D | Controller dispatch can be tested independently with mock scripts. Separates integration bugs from behavior bugs | Do both together (faster total but harder to isolate bugs) |
| Synergies as Phase E (not bundled with D) | Synergies add complexity on top of already-complex scripts. Separating allows stable scripts first, then add synergy layer | Bundle synergies into Phase D (fewer rounds but higher per-round risk) |
| Audio conditional (not core) | No audio assets secured after 5+ rounds. Cannot commit to feature without resources | Commit to audio and risk scope delay |
| Daily Challenge deferred to v1.1.2 | Seeded RNG is high-risk (must ensure deterministic behavior across platforms). Not suitable for first content expansion | Include in v1.1.0 (scope creep risk) |
| No new stage in v1.1.0 | New stage needs dedicated spawner config + visual design + balance testing (~300 lines). Better suited for a version focused on content | Add as stretch (dilutes weapon behavior focus) |
