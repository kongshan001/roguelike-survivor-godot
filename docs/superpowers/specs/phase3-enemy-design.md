# Phase 3 设计规格：7 种敌人 + Boss 三阶段

## 设计概述

参考 H5 config.js 的 7 种敌人设计，扩展 Boss 为三阶段行为，实现 5 段波次进度生成。新增远程弹幕（骷髅/精英骷髅）、相位转移（幽灵）、分裂（分裂者）三种特殊行为。

## Boss 三阶段设计

**基础数值**：HP=200, 速度=30, 伤害=2, 尺寸=32, 颜色=Color(0.96,0.26,0.21)

| 阶段 | HP 阈值 | 行为 | 弹幕 |
|------|---------|------|------|
| Phase 1 | 100%→66% (200→132) | 纯追踪玩家 | 无 |
| Phase 2 | 66%→33% (132→66) | 追踪+周期冲锋（速度×3, 0.8s） | 冲锋结束时 8 方向弹幕 |
| Phase 3 | 33%→0% (66→0) | 速度+50% | 持续 16 方向螺旋弹幕（CD=1.5s） |

## 波次进度（参考 H5 WAVE_PROGRESS）

| 阶段 | 时间 | 新增敌人 | 提示 |
|------|------|----------|------|
| 开局 | 0s | 僵尸 | 🟢 |
| 中期 | 120s | +蝙蝠 | 🟡 |
| 后期 | 180s | +骷髅、幽灵 | 🟠 |
| 精英期 | 210s | +精英骷髅 | 🔴 |
| Boss期 | 270s | Boss 生成 | 💀 |

## 无尽模式（参考 H5 ENDLESS）

- Boss 每 240s 生成一次
- Boss HP 每次递增 ×1.5，速度递增 ×1.1
- 敌人 HP 每分钟 +10%，速度每分钟 +5%
- 最大敌人数：100
- Boss 击杀奖励：50 金币

## 决策记录

- **为什么 Boss 用三阶段**：调研发现品类头部游戏（VS、Holocure）都使用多阶段 Boss，H5 单阶段较平淡
- **为什么远程用单发/3方向**：H5 原设计，骷髅单发压力适中，精英 3 方向制造扇形威胁
- **为什么幽灵用瞬移**：相位转移增加不可预测性，比纯闪烁更有战术意义
- **数值来源**：全部来自 H5 config.js ENEMY_TYPES，Boss 三阶段行为参考品类趋势
