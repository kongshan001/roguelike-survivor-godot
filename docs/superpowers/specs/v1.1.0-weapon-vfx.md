# v1.1.0 Weapon Visual Effects Spec: spiral_blade / pulse_ring / beam_line

**Author**: Art Agent
**Date**: 2026-04-17
**Round**: R27
**Status**: Design Spec
**Priority**: P2 LOW (v1.1.0 target)
**Context**: R26 Designer Agent completed evolved-weapon-behaviors.md defining the firing logic for frostvortex (spiral), holyshockwave (pulse), and thunderbeam (beam). This spec defines the visual appearance of the in-game weapon entities -- their colors, sizes, animation parameters, and particle effects. The Programmer Agent uses this spec alongside evolved-weapon-behaviors.md during Phase C+D implementation.

---

## 1. Visual Design Principles

All three weapons share these visual principles:

1. **ColorRect pixel art**: Consistent with existing project style. All visuals are ColorRect-based.
2. **Alpha decay**: Particle alpha decreases linearly over lifetime (0.15-0.3s).
3. **Weapon identity colors**: Each weapon uses its WeaponData.color and palette-matched accent colors from art-log.md R26.
4. **Layer compliance**: All projectile visuals use z_index = 5 (same as existing weapons in weapon_fire.gd).
5. **Screen-space bounds**: Effects that exceed the viewport are clipped naturally by Godot rendering.

## 3. Holy Shockwave (pulse) -- Pulse Ring Visual

### 3.1 Color Palette

| Center flash | Color(1.0, 1.0, 1.0) #FFFFFF | PALETTE["white"] |
| Burn dot (on enemy) | Color(1.0, 0.4, 0.1) #FF661A fire | Fire DOT indicator |
| Background ring | Color(0.05, 0.05, 0.1, 0.0) | Transparent -- ring is hollow |

### 3.2 Ring Visual Specification

The pulse ring is a hollow expanding circle. In pixel art, approximated as 16-20 small ColorRect segments arranged in a circle:

| Phase | Radius | Ring Width | Color | Alpha |
|---|---|---|---|---|
| Spawn (t=0) | 0 px | N/A | N/A | N/A (invisible) |
| Expanding (t=0-0.3s) | 0 -> 200 px | 12 px | Lerp from inner fire to outer gold | 0.9 -> 0.3 |
| Peak (t=0.3s) | 200 px | 12 px | Gold | 0.3 -> 0.0 (fade) |
| Fade (t=0.3-0.5s) | 200 px | 12 px -> 0 px | Gold fading | 0.3 -> 0.0 |

### 3.3 Ring Construction (Segment Approach)

For pixel art style, construct the ring from 16 small rectangles:

```
# Each segment: ColorRect 3x3 px
# Arranged at angle i/16 * TAU at current radius
# Repositioned each frame during expansion tween
# Color transitions:
#   - Inner radius (0-100px): fire_orange #FF4500
#   - Middle (100-150px): blaze_orange #FF8C00
#   - Outer (150-200px): gold #FFD700
```

Alternative (simpler implementation): Single large ColorRect with circular approximation using 8-12 segments.

## 6. Node Structure Summary

### 6.1 SpiralBlade (scripts/weapons/spiral_blade.gd)

```
SpiralBlade (Node2D, z_index=5)
  +-- ColorRect "Hub" (4x4, Color(0.3, 0.7, 1.0))
  +-- Area2D "BladeArea" (collision_layer=Layer3)
  |    +-- CollisionShape2D (CircleShape2D, radius=5.0)
  +-- ColorRect "Blade0" (5x12, Color(0.53, 0.87, 1.0)) x 6
```

### 6.2 PulseRing (scripts/weapons/pulse_ring.gd)

```
PulseRing (Node2D, z_index=5)
  +-- ColorRect "CenterBurst" (8x8, white, fades)
  +-- ColorRect "Segment0" (3x3, gold) x 16
  +-- Area2D "RingArea" (collision_layer=Layer3)
       +-- CollisionShape2D (CircleShape2D, radius=grows)
```

### 6.3 BeamLine (scripts/weapons/beam_line.gd)

```
BeamLine (Node2D, z_index=5)
  +-- ColorRect "BeamCore" (1200x2, Color(1.0, 0.84, 0.0))
  +-- ColorRect "CenterLine" (1200x1, white)
  +-- ColorRect "EdgeGlowL" (1200x1, Color(0.3, 0.5, 1.0, 0.5))
  +-- ColorRect "EdgeGlowR" (1200x1, Color(0.3, 0.5, 1.0, 0.5))
  +-- ColorRect "StartPoint" (3x3, white)
  +-- ColorRect "EndPoint" (3x3, white)
  +-- Area2D "BeamArea" (collision_layer=Layer3)
  |    +-- CollisionShape2D (RectangleShape2D, 12x1200)
  +-- Timer "TickTimer" (0.3s, loops)
```

## 7. Performance Budget

| Weapon | Max ColorRects per instance | Particle budget | Notes |
|---|---|---|---|
| frostvortex (spiral) | 7 (1 hub + 6 blades) + Area2D | 2/hit, max ~12/s | Persistent, always active |
| holyshockwave (pulse) | 16 segments + 1 burst + Area2D | 3/hit, max ~9/pulse | Transient, 0.5s lifetime |
| thunderbeam (beam) | 6 static + 1 spark pool + Area2D | 1/0.1s = 10/activation | Transient, 1.0s lifetime |

Total concurrent visual nodes (worst case, all 3 active):
- Spiral: 7 persistent
- Pulse: ~18 transient (0.5s every 2.5s)
- Beam: ~16 transient (1.0s every 2.5s)
- Particles: ~30 (well within MAX_PARTICLES=60)

## 8. Visual Differentiation Summary

All three new weapons have visually distinct silhouettes:

| Weapon | Shape | Primary Visual | Motion | Distinguishing Feature |
|---|---|---|---|---|
| frostvortex | Radial | 6 rotating blades | Continuous spiral | Ice blue, expanding/contracting radius |
| holyshockwave | Circular ring | Expanding hollow ring | Periodic burst | Gold-to-fire gradient, center flash |
| thunderbeam | Linear | Long horizontal line | Point-and-shoot | Electric yellow with blue edge glow |

None of these overlap visually with existing weapon effects (projectiles, boomerangs, orbit circles, aura zones).

## 9. Implementation Notes for Programmer Agent

1. All Color values in this spec use Godot Color(r,g,b) format (0.0-1.0 range).
2. Particle alpha decay uses `tween_property(rect, "color:a", 0.0, lifetime)` pattern from hit_feedback.gd.
3. SpiralBlade blade positions calculated per-frame in `_process()` -- no tween for position, only for Frostbite Loop flash.
4. PulseRing segments repositioned via tween on a single "scale" property applied to the parent Node2D.
5. BeamLine is rotated toward target enemy; all child ColorRects are axis-aligned relative to the beam direction.
6. Chain lightning zigzag uses random perpendicular offsets -- re-seed with different random values each time for natural variation.
