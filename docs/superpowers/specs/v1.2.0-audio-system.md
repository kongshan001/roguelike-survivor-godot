## 1. Executive Summary

v1.2.0 音频系统是项目从"视觉完整"走向"体验完整"的关键一步。当前项目有 2239 个测试全通过，v1.1.0 核心功能完成度约 90%，但音频系统完成度为 0%。本规格定义了 AudioManager autoload 单例架构、音频总线布局、BGM/SFX 完整列表、音量控制机制和音频资源格式规范。

**核心目标**: 让游戏从"无声演示"变成"有声可玩"，覆盖 5 首 BGM + 30+ 个 SFX 触发点，提供音量控制和静音开关。

## 2. Pre-requisite Research

### 2.1 Genre Audio Analysis

| Game | BGM Approach | SFX Density | Volume Control | Reference Value |
|------|-------------|-------------|----------------|-----------------|
| Vampire Survivors | 1 looping BGM per stage, crossfade on stage change | ~50 SFX (weapon hits, level-up, chest, death) | Master + BGM + SFX sliders in options | **HIGH** -- same genre, proven pattern |
| Brotato | Wave-based BGM intensity, shop music | ~40 SFX, layered per weapon type | In-game pause menu sliders | **MEDIUM** -- wave-based audio intensity is interesting |
| Holocure | Character-specific BGM, boss music transition | ~60 SFX with pitch variation | Options menu with 4 sliders | **MEDIUM** -- character-specific BGM adds personality |
| Magic Survival | Minimal ambient BGM, SFX-heavy | ~20 SFX, simple approach | Single volume slider | **LOW** -- too minimal for our scope |

**Key findings**:
1. **Bus layout standard**: Master / BGM / SFX three-bus layout is the industry standard for this genre. Adding a UI bus for button/click sounds is common but optional for v1.2.0.
2. **Crossfade transitions**: BGM crossfade (1-2 seconds) is expected during stage transitions. Vampire Survivors uses ~1.5s fade.
3. **Pitch randomization**: SFX pitch variation (+/- 5-10%) on repetitive sounds (weapon hits, enemy deaths) prevents auditory fatigue. This is a must-have, not a nice-to-have.
4. **Audio pooling**: Frequently triggered SFX (hit, pickup) need AudioStreamPlayer pooling (3-5 instances) to avoid cutoff and glitching.

### 2.2 Godot 4.6 Audio Architecture

Godot 4.6 provides:
- **AudioBusLayout** resource for bus configuration
- **AudioStreamPlayer** for 2D positional audio
- **AudioStreamPlayer2D** for position-aware SFX
- **AudioStreamOGGVorbis** for compressed BGM (streaming)
- **AudioStreamWAV** for short SFX (in-memory)
- **AudioServer** for bus volume control (linear_to_db conversion)

For this project:
- BGM: `AudioStreamPlayer` (non-positional, always center)
- SFX (UI): `AudioStreamPlayer` (non-positional)
- SFX (gameplay): `AudioStreamPlayer2D` (positional, distance-attenuated) -- optional for v1.2.0, start non-positional

## 3. AudioManager Architecture

### 3.1 Autoload Singleton

**Decision**: AudioManager will be an autoload singleton (like GameManager, SynergyManager).

**Why autoload instead of scene-level**: Audio must persist across scene transitions (main -> character_select -> difficulty_select -> arena). Scene-level managers would need to be re-initialized on every scene load, and BGM crossfade during transitions would be impossible without a persistent node.

**File**: `scripts/autoload/audio_manager.gd`
**Estimated lines**: ~150

### 3.2 Node Structure

```
AudioManager (Node, autoload)
  +-- AudioStreamPlayer "bgm_player" (BGM bus)
  +-- AudioStreamPlayer "bgm_fade_player" (BGM bus, for crossfade)
  +-- AudioStreamPlayer "sfx_player_1" (SFX bus)
  +-- AudioStreamPlayer "sfx_player_2" (SFX bus)
  +-- AudioStreamPlayer "sfx_player_3" (SFX bus)
  +-- AudioStreamPlayer "sfx_player_4" (SFX bus)
  +-- AudioStreamPlayer "ui_player" (UI bus)
  +-- Timer "fade_timer" (for BGM crossfade timing)
```

**Pool size = 4 SFX players**: Based on genre analysis, 4 simultaneous SFX is sufficient for most gameplay scenarios. In extreme cases (dense wave, multiple weapon hits), excess sounds are silently dropped rather than interrupting active sounds.

### 3.3 Audio Bus Layout

```
Master (bus 0)
  +-- BGM (bus 1)   -- Background music, streamed OGG
  +-- SFX (bus 2)   -- Gameplay sound effects, in-memory WAV
  +-- UI  (bus 3)   -- UI clicks, notifications, in-memory WAV
```

**Bus layout file**: `assets/audio/bus_layout.tres` (created in Godot editor or manually)

**Default volumes**:

| Bus | Linear (0-1) | dB | Notes |
|-----|-------------|-----|-------|
| Master | 1.0 | 0.0 | Full volume, player-controlled |
| BGM | 0.6 | -4.4 dB | Slightly lower than SFX -- music should support, not dominate |
| SFX | 0.8 | -1.9 dB | Primary gameplay feedback |
| UI | 0.7 | -3.1 dB | Slightly lower than gameplay SFX |

### 3.4 Volume Control

Volume uses a 0-100 integer scale (user-facing) mapped to dB (Godot internal).

**Mapping formula**:
```gdscript
func linear_to_volume_db(value: int) -> float:
    # value: 0-100 (user-facing)
    # 0 = -40 dB (effectively mute), 100 = 0 dB (full)
    if value <= 0:
        return -40.0
    var linear: float = value / 100.0
    return linear_to_db(linear)
```

**Volume storage**: Persisted via SaveManager in save_data["settings"]["volumes"].

```gdscript
var _volumes: Dictionary = {
    "master": 80,  # Default 80/100
    "bgm": 60,     # Default 60/100
    "sfx": 80,     # Default 80/100
    "ui": 70       # Default 70/100
}
```

### 3.5 Fade In/Out Mechanism

**BGM Crossfade** (transition between two BGM tracks):

```gdscript
func play_bgm(stream: AudioStream, fade_time: float = 1.5) -> void:
    # If already playing, crossfade to new track
    if _bgm_player.playing:
        _bgm_fade_player.stream = _bgm_player.stream
        _bgm_fade_player.play(_bgm_player.get_playback_position())
        _bgm_fade_player.volume_db = _bgm_player.volume_db
        # Fade out old player
        var tween: Tween = create_tween()
        tween.tween_property(_bgm_fade_player, "volume_db", -40.0, fade_time)
        tween.tween_callback(_bgm_fade_player.stop)
    # Fade in new player
    _bgm_player.stream = stream
    _bgm_player.volume_db = -40.0
    _bgm_player.play()
    var tween: Tween = create_tween()
    tween.tween_property(_bgm_player, "volume_db", linear_to_volume_db(_volumes["bgm"]), fade_time)
```

**Fade times**:

| Transition | Fade Time | Notes |
|-----------|-----------|-------|
| Title -> Character Select | 1.0s | Quick, upbeat transition |
| Character Select -> Arena | 1.5s | Build anticipation |
| Arena Standard -> Boss | 0.5s | Quick intensity shift |
| Boss -> Victory | 2.0s | Slow, triumphant fade |
| Any -> Game Over | 1.0s | Somber fade |

### 3.6 Mute Toggle

```gdscript
var _muted: bool = false

func toggle_mute() -> void:
    _muted = not _muted
    AudioServer.set_bus_mute(AudioServer.get_bus_index("Master"), _muted)

func is_muted() -> bool:
    return _muted
```

**Toggle binding**: F1 key or UI button in pause menu.

**Pitch variation implementation**:
```gdscript
func play_sfx(id: String, pitch_variation: float = 0.0) -> void:
    var player: AudioStreamPlayer = _get_available_sfx_player()
    if not player:
        return  # Pool exhausted, silently drop
    player.stream = _sfx_cache[id]
    player.pitch_scale = 1.0 + randf_range(-pitch_variation, pitch_variation)
    player.play()
```

**Throttle for enemy_hurt**: In dense waves (50+ enemies), damage ticks can trigger 20+ hurt sounds per second. Throttle to max 5 plays/second to prevent audio chaos.

```gdscript
var _hurt_sfx_cooldown: float = 0.0
const HURT_SFX_MIN_INTERVAL: float = 0.2  # 5 per second max

func _process(delta: float) -> void:
    _hurt_sfx_cooldown = maxf(0.0, _hurt_sfx_cooldown - delta)
```

## 6. Audio Resource Format Specification

### 6.1 Format Requirements

| Category | Format | Sample Rate | Bit Depth | Channels | Max File Size | Notes |
|----------|--------|-------------|-----------|----------|---------------|-------|
| BGM | OGG Vorbis | 44100 Hz | 16-bit | Stereo | 2 MB per track | Loop-enabled, streaming playback |
| SFX | WAV (PCM) | 44100 Hz | 16-bit | Mono | 100 KB per file | In-memory, low latency |

**Why OGG for BGM**: Godot streams OGG files from disk, avoiding large memory allocation for long music tracks. OGG at quality 5 (~128 kbps) provides good quality at small file sizes. A 120-second stereo BGM at 128 kbps = ~1.9 MB.

**Why WAV for SFX**: SFX are short (< 0.5s) and triggered frequently. WAV provides the lowest playback latency (no decode overhead). Mono because SFX do not need stereo imaging -- the audio system handles spatialization if needed.

### 6.2 File Size Budget

| Category | Count | Est. Size Each | Total |
|----------|-------|---------------|-------|
| BGM | 6 tracks | 1.5 MB | 9.0 MB |
| SFX (player) | 5 | 30 KB | 150 KB |
| SFX (weapon) | 12 | 25 KB | 300 KB |
| SFX (enemy) | 4 | 30 KB | 120 KB |
| SFX (UI) | 6 | 15 KB | 90 KB |
| SFX (environment) | 6 | 25 KB | 150 KB |
| **Total** | **39 files** | -- | **~9.8 MB** |

func _generate_placeholder_sfx(frequency: float, duration: float, wave_type: String = "square") -> AudioStream:
    var stream: AudioStreamGenerator = AudioStreamGenerator.new()
    stream.mix_rate = 44100
    # Generate a simple beep/tone as placeholder
    # This allows testing the full audio pipeline without real assets

This approach:
- Validates the AudioManager architecture independently of asset availability
- Provides audible feedback during development

## 12. Decision Records

| Decision | Why | Alternative Considered |
|----------|-----|----------------------|
| Autoload singleton (not scene-level) | Audio must persist across scene transitions; BGM crossfade requires a persistent node | Scene-level AudioManager in each scene (duplicated state, no crossfade between scenes) |
| 4-bus layout (Master/BGM/SFX/UI) | Industry standard for this genre; separates concerns for independent volume control | 3-bus (no UI bus) -- UI sounds mixed with SFX is acceptable for v1.2.0 but limits future control |
| SFX pool of 4 players | Genre analysis shows 4 simultaneous SFX sufficient; 3 is too few for dense waves, 8 wastes resources | Dynamic pool (create on demand) -- risk of unbounded memory growth during dense waves |
| WAV for SFX, OGG for BGM | WAV = lowest latency for short sounds; OGG = streaming for long music (lower memory) | OGG for everything -- higher decode overhead per SFX trigger, measurable on slow devices |
| Mono SFX | SFX do not need stereo; mono is half the file size and memory; spatialization handled by AudioServer if needed | Stereo SFX -- double file size with no audible benefit for this pixel-art game |
| Pitch variation on repetitive SFX | Prevents auditory fatigue for weapon hits and enemy deaths (+/- 5-12% variation is industry standard) | No variation -- every hit sounds identical, becomes grating after 100+ repetitions |
| Hurt SFX throttle (5/sec max) | Dense waves can trigger 20+ damage events per second; unthrottled SFX creates audio cacophony | No throttle -- audio becomes noise, defeats the purpose of sound feedback |
| Default BGM volume lower than SFX | Music should support gameplay, not dominate; SFX carries critical gameplay information (hits, deaths) | Equal BGM/SFX volume -- music masks important gameplay audio cues |
| Placeholder via AudioStreamGenerator | Allows full pipeline testing without blocking on external assets; sounds can be swapped with zero code changes | Block on real assets -- audio system cannot be tested until assets arrive |
| Non-positional SFX (AudioStreamPlayer) | Simpler implementation; this game's camera is always centered on player so spatial audio adds minimal value | AudioStreamPlayer2D for all SFX -- more complex, negligible benefit for centered camera |
| Volume range 0-100 integer | Familiar to players (percentage scale); Godot internally uses dB, conversion is handled by AudioManager | 0-1 float scale -- less intuitive for UI display; dB scale -- confusing for non-technical users |
