# v1.0.1 Feature Priority Assessment

**Author**: Designer Agent
**Date**: 2026-04-17
**Round**: R17
**Status**: Design Spec
**Context**: v1.0.0 released (1191 tests, 94.2/100 score). R16 identified 7 friction points. Reviewer flagged 4 P1 issues.

---

## 1. Assessment Criteria

Each item is evaluated on three dimensions:

- **User Value (1-5)**: How much does this improve the player's first-session experience, retention, or satisfaction? 5 = critical for new player onboarding, 1 = nice-to-have polish.
- **Implementation Cost (1-5, 5 = low cost)**: How many lines of code, new files, or integration complexity? 5 = under 50 lines with no new files, 1 = multi-day effort with new systems.
- **Risk (High/Medium/Low)**: Chance of introducing regressions, breaking existing tests, or requiring follow-up fixes.

## 2. Item-by-Item Assessment

### Item 1: Tutorial System (tutorial-system.md)

| Dimension | Score | Rationale |
|---|---|---|
| User Value | 5 | The game has 3 core interactions (move+dash, upgrade selection, skill activation) with zero explanation. New players face 7 HUD elements and auto-firing weapons with no guidance. This is the single highest-impact UX improvement. |
| Implementation Cost | 4 | 1 new script (~120 lines) + 4 small modifications (save_manager +4, arena +8, hud +15, hud_skill_button +12). Total ~159 lines. Design spec fully specified. |
| Risk | Low | Purely additive UI layer. No game logic changes. Tutorial state persisted via SaveManager (existing infrastructure). |

**Recommended Version**: v1.0.1

---

### Item 2: Performance Optimization -- get_nodes_in_group Cache

| Dimension | Score | Rationale |
|---|---|---|
| User Value | 3 | Invisible to players unless they experience frame drops at high enemy counts (70-100). Preventative improvement. |
| Implementation Cost | 3 | Requires refactoring 9 files that call get_nodes_in_group. Needs a shared cache system (likely in game_manager.gd or a new utility). Moderate integration complexity. |
| Risk | Medium | Changing how enemy/player lists are accessed affects 9 scripts. If cache invalidation is wrong, enemies could become invisible to weapons or vice versa. |

**Recommended Version**: v1.0.1 (if regression risk is accepted) or v1.1.0 (safer)

---

### Item 3: Character Animation Frame Integration

| Dimension | Score | Rationale |
|---|---|---|
| User Value | 2 | Visual polish only. Characters currently display as static sprites (mage.png, warrior.png, ranger.png). Adding walk/idle animation frames improves juice but does not affect gameplay. |
| Implementation Cost | 2 | Requires Art Agent to generate animation frames (at minimum idle + walk cycles for 3 characters = 6+ sprite sheets), then Programmer to integrate AnimatedSprite2D nodes into player.tscn. |
| Risk | Low | Additive visual change, no game logic impact. |

**Recommended Version**: v1.2.0 (content update)

---

### Item 4: UI Animations (Tween) -- Wave Banner / Upgrade Panel / Damage Numbers / Gold

| Dimension | Score | Rationale |
|---|---|---|
| User Value | 4 | R16 identified friction point 5 ("wave transitions lack visual buffer"). Wave banners with slide-in animation create rhythm and ceremony. Damage numbers provide combat feedback. Upgrade panel animations make card selection feel responsive. |
| Implementation Cost | 3 | Wave banner already has a complete VFX spec (wave-transition-vfx.md). Implementing all 4 animation categories is ~150-200 lines across hud.gd and potentially a new ui_animations.gd helper. |
| Risk | Low | Tween-based animations are purely visual. If a Tween fails, the element just appears without animation (graceful degradation). |

**Recommended Version**: v1.0.1 (wave banner at minimum), v1.1.0 (full suite)

---

### Item 5: Thunderang / Blazerang Special Attack Patterns

| Dimension | Score | Rationale |
|---|---|---|
| User Value | 3 | P1-1 and P1-2 from release-readiness review. These evolved weapons feel underpowered compared to H5 definitions (thunderang missing lightning chain with 0.4 chance, 2 targets, 2 chains; blazerang missing flame trail with 2 DPS, 2.5s burn). |
| Implementation Cost | 4 | H5 config.js already defines exact parameters. thunderang needs chain lightning logic (similar to existing lightning weapon, ~40 lines). blazerang needs trail spawning (~30 lines). Both fit within weapon_boomerang_fire.gd. |
| Risk | Low | Scoped to evolved weapon behavior only. Base boomerang unaffected. New behavior only activates when is_evolved=true. |

**Recommended Version**: v1.0.1

---

### Item 6: weapon_fire.gd Unused Constants Cleanup

| Dimension | Score | Rationale |
|---|---|---|
| User Value | 1 | Technical debt cleanup. BUG-272. 4 unused constants: HOLYWATER_LV3_SLOW_PCT (duplicate of projectile.gd:23), LIGHTNING_LV3_CHAIN_BONUS (defined but never referenced), BURN_DPS and BURN_DURATION (base 2.0 values shadowed by FIRESTAFF_LV3 variants). |
| Implementation Cost | 5 | Delete 4 lines. Run tests to confirm no references were missed. |
| Risk | Low | Grep confirms the constants are unused in weapon_fire.gd. The used versions exist in other files (projectile.gd has its own HOLYWATER_LV3_SLOW_PCT). |

**Recommended Version**: v1.0.1 (trivial cleanup)

### Item 7: Character Select Page Enhancement

| Dimension | Score | Rationale |
|---|---|---|
| User Value | 4 | R16 friction point 1 ("character selection lacks information"). Current cards show name/HP/speed/one-line-desc/passive. Missing: initial weapon behavior, skill description, difficulty recommendation. Players choose characters blind. |

| get_nodes_in_group Cache in v1.0.1 with Medium risk | The 9-file refactor adds regression risk. Recommend Programmer implement with thorough test coverage before merge. If tests fail, bump to v1.1.0. | Defer to v1.1.0 unconditionally (safer but leaves known performance gap) |
| Character Animation Frames deferred to v1.2.0 | Visual-only improvement with Art dependency (needs 6+ sprite sheets). Not blocking any gameplay issue. | Include in v1.0.1 (blocks hotfix on Art deliverables) |
| Full UI Animation Suite split across v1.0.1 and v1.1.0 | Wave banner is most impactful (R16 friction point 5) and already has a complete VFX spec. Damage numbers and gold popups are lower priority and can wait. | Implement all animations in v1.0.1 (scope creep for hotfix) |

## 5. v1.0.1 Acceptance Criteria

The v1.0.1 hotfix is considered complete when:

1. All 1191 existing tests still pass (zero regressions)
2. Tutorial system triggers on first game and never repeats
3. Character select cards show: skill name + description, weapon name + behavior, recommended difficulty
4. thunderang fires lightning chains (chance 0.4, targets 2, chains 2, decay 0.5)
5. blazerang leaves flame trails (trailInterval 20, trailDur 1.5, trailDps 2, burnDur 2.5)
6. Wave start banner slides in from top with wave-specific color
7. 4 unused constants removed from weapon_fire.gd
8. get_nodes_in_group call frequency reduced by at least 50%
9. Total test count increases by at least 20 (new tests for tutorial, special attacks, wave banner)
