v1.0.3 是 v1.0.2 之后的收尾与扩展版本，聚焦三个方向：(1) 完成 v1.0.2 遗留的 UI 工作（精通徽章、暂停面板、新手引导 3 步），(2) 启动音频系统（需评估外部资源可行性），(3) 清理技术债务以释放 97+ 分数的空间。本版本的目标是从"功能完整"迈向"体验完整"。

## 4. Feature Candidates

### 4.1 Weapon Mastery UI Integration [Priority 1]

**Priority**: P1 HIGH
**Est. Lines**: ~107 (80 new module + 27 hud.gd additions)
**Blocker**: None. Backend (SaveManager mastery functions) deployed in R20.
**Spec**: `docs/superpowers/specs/weapon-mastery-ui.md`

**Three-layer UI**:

| Layer | Component | Description |
|---|---|---|
| HUD Badge | 6x6 ColorRect | Weapon slot bottom-right corner. Tier 0 hidden, 1=bronze, 2=silver, 3=gold, 4=diamond (pulsing) |
| Level-up Toast | Reuse hud_toast.gd | "Knife mastery: Expert!" on tier up. Tier 3-4 triggers full-screen color flash (0.4s) |
| Pause Panel | hud_mastery_panel.gd (~80 lines) | 300x420px, 7 rows, progress bars (80px), kill counts, bonus percentages |

**Integration points**:
- SaveManager emits `mastery_tier_up(weapon_id, tier)` signal (new)
- hud.gd connects signal, creates HudMasteryPanel RefCounted module
- Pause state: when `get_tree().paused`, show mastery panel overlay

**Why first**: Mastery backend has been deployed since R20 with no UI. Players cannot see their progress. This is the highest-value incomplete item from v1.0.2.

### 4.2 Tutorial Steps 6-8 [Priority 2]

**Priority**: P2 MEDIUM
**Est. Lines**: ~40
**Blocker**: None. TutorialManager exists since R17.
**Spec**: `docs/superpowers/specs/v1.0.2-roadmap.md` Section 4.5

**Three new steps**:

| Step | Trigger | Content |
|---|---|---|
| 6 | Player has 2 weapons | "Weapons can combine at Lv3 -- check evolution recipes!" |
| 7 | Combo >= 5 | "Combo kills give bonus XP! Keep killing to maintain your streak." |
| 8 | First synergy triggers | "Synergy activated! Some weapon+passive combos create powerful effects." |

**Implementation**: Extend TutorialManager._ready() with 3 new step definitions. Each step has its own trigger condition checked in _process() or via signal.

**Why second**: Lowest cost with measurable new-player impact. Tutorial currently covers movement/dash/weapons/upgrade/skill (5 steps) but misses three key mid-game mechanics that confuse new players.

### 4.3 Audio System [Priority 3]

**Priority**: P1 HIGH (impact) but BLOCKED by external assets
**Est. Lines**: ~400 (code only)
**Spec**: `docs/superpowers/specs/v1.0.2-roadmap.md` Section 4.4

**External Resource Assessment**:

| Asset Type | Options | Cost | Quality | Recommendation |
|---|---|---|---|---|
| BGM (5 tracks) | Royalty-free (freesound.org, opengameart.org) | Free | Medium | Use as placeholder, upgrade later |
| BGM (5 tracks) | AI-generated (Suno, Udio) | Low | Medium-High | Best cost/quality ratio for indie |
| BGM (5 tracks) | Commissioned | High | High | v1.1.0 investment |
| SFX (20+ clips) | Godot built-in generators | Free | Low | Prototype only |
| SFX (20+ clips) | Royalty-free (kenney.nl) | Free | Medium | Recommended for v1.0.3 |
| SFX (20+ clips) | Foley/recorded | Medium | High | v1.1.0 investment |

**SFX Categories (20+ clips)**:
- Weapon fire (7): knife_throw, holywater_spin, lightning_strike, bible_flip, firestaff_burst, frostaura_hum, boomerang_throw
- Combat (4): enemy_hit, enemy_death, player_hurt, crit_impact
- UI (4): level_up, coin_pickup, xp_collect, toast_notification
- System (3): dash, skill_activate, evolution_fanfare
- Meta (2): menu_music_start, game_over_sting

**Implementation Plan** (if assets secured):

| Phase | Content | Lines | Risk |
|---|---|---|---|
| Phase A | AudioManager autoload + BGM crossfade | ~120 | Low |
| Phase B | SFX trigger points (15+ scripts) | ~200 | Low (additive) |
| Phase C | Volume controls + mute toggle | ~80 | Low |

**AudioManager Architecture**:
```
autoload/
  audio_manager.gd (~120 lines)
    - AudioStreamPlayer (BGM, loop)
    - Array of AudioStreamPlayer2D (SFX pool, 8 slots)
    - crossfade(old, new, duration)
    - play_sfx(id, position)
    - set_volume(bus, value)
    - toggle_mute()
```

**Decision point**: If no audio assets can be secured within 2 days, defer Audio System to v1.1.0 and focus on items 4.1 and 4.2 only.

### 4.4 Register 3 Unregistered Evolved Weapons [Priority 4]

**Priority**: P2 LOW
**Est. Lines**: ~60
**Blocker**: None.
**Spec**: Referenced in designer-log "3种未注册进化武器"

**Weapons to register**:

| Weapon | Type | Parents | Status |
|---|---|---|---|
| frostvortex | Aura | frostaura + holywater | Defined in spec but not in upgrade_pool.gd |
| holyshockwave | Cone | firestaff + holywater | Defined in spec but not in upgrade_pool.gd |
| thunderbeam | Lightning | lightning + firestaff | Defined in spec but not in upgrade_pool.gd |

**Impact**: Currently these 3 evolution paths silently fail (no match in weapon_registry). Players who build the correct dual-Lv3 combo get no reward. Low impact because the other 6 evolution paths work correctly.

**Why not higher**: 9 evolved weapons already work. These 3 are Godot-original content not present in H5. No player-facing bug exists -- the evolutions simply do not occur.
